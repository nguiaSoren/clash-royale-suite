Novelty analysis — deck synergy & composition model

What's new, what exists, and why this approach is distinct

This document positions the deck synergy module against existing deck-building tools, game AI research, and related work in combinatorial optimization. The analysis covers novelty in the Clash Royale context and in the broader game AI / multi-agent systems space.

1. Existing deck-building approaches landscape

Every public deck recommendation system for Clash Royale operates on the same principle: observe what humans play, rank by win rate. This is the statistical approach.

SystemMethodCan evaluate unplayed decks?Explains why?
RoyaleAPIWin rate aggregation from top ladderNoNo
DeckShopHeuristic scoring (elixir curve, roles)Partially (heuristic)Partially
StatsRoyaleUsage + win rate statisticsNoNo
Community tier listsExpert opinion + meta observationNoSubjective

These systems know that certain decks win. They don't know why. They can't evaluate a deck nobody has played. They can't predict how a brand-new card will fit into existing archetypes. And they break every time the meta shifts — they have to wait for humans to discover the new winning compositions before they can report them.

DeckShop is the closest to a structural approach — it scores decks using heuristic rules (needs a win condition, needs air defense, elixir curve should be balanced). But its synergy assessment is rule-based, not learned from mechanical interactions. It doesn't simulate.

2. What this project does differently novelty

This module takes a fundamentally different approach: learn why cards synergize from their mechanical attributes, verified through exhaustive simulation, then compose novel decks from discovered synergy clusters.

2.1 Simulation-verified synergy scoring

No one in game AI — not just Clash Royale — measures unit synergy by exhaustively simulating pairwise and triple interactions in a deterministic engine, then training embedding models on the results. The standard approaches are either pure statistics (observe win rates) or pure RL (let the agent figure out everything from scratch). This is a middle path: measure mechanical interactions first, embed them into learned representations, then use those representations to compose.

The deterministic simulator makes this possible. C(178,2) = 15,753 pairwise interactions run in seconds. C(178,3) = 929,476 triple interactions run in minutes. No other Clash Royale project has a simulator that could produce this data.

2.2 Multi-order synergy with cluster discovery

Pairwise synergy is well-studied in recommendation systems. What's distinct here is the three-level composition pipeline:

Pairwise scores (15K)  ──┐
                         ├──→  Synergy clusters (50-100 groups of 2-4 cards)
Triple scores (929K)   ──┘              │
                                        ├──→  Deck = 2-3 clusters assembled
Inter-cluster scores   ─────────────────┘     (166K combos, not 4.5 trillion)
Three-level composition: card interactions → synergy clusters → deck assembly. Each level reduces the search space by orders of magnitude.

This mirrors how expert players actually build decks — they don't pick 8 cards independently. They pick a push package (Golem + Baby Dragon + Tornado), a defense package (Inferno Dragon + Tombstone), and fill in cycle/spells (Zap + Skeletons + Poison). The cluster discovery formalizes this intuition with simulation-backed scores, and the inter-cluster synergy measurement captures the emergent interactions between packages.

The hierarchical structure — clusters within decks — is conceptually similar to hierarchical RL (high-level policy selects sub-goals, low-level policy executes), but applied to composition rather than action selection. The high-level policy assembles clusters into a deck; the low-level RL agent plays the deck. They improve each other iteratively.

2.3 Generative, not statistical

The critical distinction: this system can compose decks that have never been played by any human. It reasons from card attributes (the 35-field mechanical vector extracted from 297 JSON fields) rather than from observed play patterns. When Supercell releases a new card, the system reads its JSON attributes, computes its synergy with every existing card, places it into existing clusters or forms new ones, and evaluates deck compositions — all before a single player has tried it.

This is the difference between "these decks have won" and "these cards should work together because their mechanics complement each other."

2.4 Data-driven role derivation

Roles (tank, splash, win condition, anti-air) are not hardcoded labels. They are derived from the same JSON fields used for the attribute vector: target_only_buildings=True → win condition, hitpoints > 2000 → tank, area_damage_radius > 0 → splash. A single card can hold multiple roles (Mega Knight = tank + splash + dasher). This makes the system fully data-driven — new cards get classified automatically, and the role constraints in deck composition update without manual intervention.

3. Related work beyond Clash Royale broader context

The deck synergy problem — finding optimal compositions of heterogeneous agents with attribute vectors — appears across multiple domains under different names.

Drug combination discoveryFind drug pairs/triples with synergistic effectsPairwise screening + statistical modelsAnalogous: exhaustive pairwise/triple simulation → embedding model → predict higher-order combinations. Different domain, same pipeline.
Recommendation systemsComplementary item bundlesCollaborative filtering (user-item matrices)This approach uses item attributes + causal simulation rather than user behavior. Cold-start capable (new items work immediately).

The closest academic parallel is combinatorial drug synergy prediction: exhaustive pairwise assays → train interaction models → predict higher-order combinations. The key difference is that drug assays are expensive (hours per pair), while game simulations are nearly free (microseconds per pair). This enables exhaustive triple measurement, which drug discovery can't afford — making the triple synergy model and cluster discovery feasible in a way they aren't in pharma.

5. Summary of novel contributions this project

Four aspects of this approach are distinct from existing work:

  • Simulation-verified synergy scoring: Exhaustive pairwise (15K) and triple (929K) interaction measurement in a deterministic engine, producing mechanically-grounded synergy scores — not statistical correlations from observed play.
  • Multi-order embedding with cluster discovery: Two separate MLPs for pairwise and triple interactions, feeding into agglomerative clustering to discover natural card groupings. Deck composition operates on clusters (166K combinations) rather than individual cards (4.5 trillion combinations).
  • Generative deck composition: The system composes decks that have never been played, reasoning from card attributes rather than historical win rates. New cards are evaluated immediately from their JSON definition — no retraining, no waiting for meta discovery.
  • Data-driven role derivation: Card roles derived from mechanical attributes, not manually labeled. Multi-role classification (a card can be tank + splash + dasher simultaneously) enables fine-grained constraint satisfaction in deck assembly.

The combination of deterministic simulation + simulation-derived synergy + cluster-based hierarchical composition is, to our knowledge, novel in the game AI literature. Individual pieces exist — pairwise interaction modeling, hierarchical policies, attribute-based embeddings — but the end-to-end pipeline connecting them through a verified game simulator has not been demonstrated.
